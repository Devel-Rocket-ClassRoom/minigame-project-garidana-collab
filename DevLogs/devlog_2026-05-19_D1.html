03

구현 상세

🎮 PlayerMovement.cs
  • 방향키 8방향 이동 — Input System 기반, InputAction으로 벡터 읽어 CharacterController 이동
  • 이동 방향 회전 — Quaternion.LookRotation으로 캐릭터가 이동 방향을 바라봄
  • 대시(D키) — 방향키 입력 방향으로 순간 이동 처리, 쿨다운 관리
  • 스프린트 — Shift 계열 입력으로 이동 속도 배속 적용
  • 중력 처리 — isGrounded 확인 후 중력 벡터 누적
  • 애니메이션 연동 — Animator 파라미터(Speed, IsDashing) 실시간 전달
📥 PlayerInputReader.cs
  • Input System 래퍼 — InputSystem_Actions 인스턴스 관리, 이벤트 구독
  • MoveInput (Vector2) — 방향키/WASD 입력값 노출
  • DashInput (bool) — D키 Dash 액션 이벤트 처리
  • SprintInput (bool) — 스프린트 상태 bool 노출
  • PlayerMovement에서 this 컴포넌트 참조로 입력 분리
🎥 PlayerCameraZoom.cs
  • 쿼터뷰 카메라 기본 셋업
  • 스크롤 입력으로 카메라 줌인/줌아웃 처리
  • 최소/최대 줌 거리 클램프
  • Cinemachine 미사용, 직접 Transform 조작 방식
🎞 PlayerController.controller
  • Idle → Walk → Sprint 상태 전이 구성
  • Dash 상태 별도 트리거 전이 추가
  • 파라미터 : Speed (float), IsDashing (bool)
  • Human Melee Animations 에셋 클립 연결
