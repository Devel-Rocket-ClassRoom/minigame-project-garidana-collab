HitStopManager와 PauseManager 역할 분리 기록

작성일: 2026-05-25 / 대상 프로젝트: 김경일의 모험 Origin / 주제: 시간 제어 책임 분리

요약

기존에는 HitStopManager가 타격 연출용 히트스톱과 옵션 메뉴/게임오버 일시정지를 모두 처리했습니다. 이번 수정에서는 전투 연출은 HitStopManager에 남기고, 메뉴와 게임오버처럼 의도적으로 게임을 멈추는 기능은 새 PauseManager로 분리했습니다.

수정 전 구조

수정 전 HitStopManager는 이름상으로는 히트스톱 매니저였지만, 실제 역할은 두 가지였습니다.

  • 플레이어 공격이 적중했을 때 HitStopManager.Request(...)로 짧은 정지 연출을 적용했습니다.
  • 옵션 메뉴와 게임오버 UI에서 HitStopManager.SetGlobalTimeScale(0f), SetGlobalTimeScale(1f)로 게임을 멈추거나 재개했습니다.

기존 책임

HitStopManager
├─ 공격 적중 히트스톱
└─ 옵션 메뉴/게임오버 일시정지

문제점

  • 클래스 이름과 실제 책임이 맞지 않았습니다.
  • 전투 연출 코드와 UI 일시정지 코드가 같은 매니저에 의존했습니다.
  • 나중에 일시정지 규칙이 늘어나면 히트스톱 코드까지 복잡해질 가능성이 있었습니다.

수정 후 구조

수정 후에는 시간 제어를 목적에 따라 나누었습니다. 짧은 타격감 연출은 HitStopManager, 사용자가 메뉴를 열거나 게임오버 상태가 되었을 때의 정지는 PauseManager가 담당합니다.

클래스 역할 주요 호출 위치
HitStopManager 공격 적중 시 아주 짧게 시간을 늦춰 타격감을 만드는 전투 연출 전용 매니저 PlayerAttack
PauseManager 옵션 메뉴, 게임오버처럼 게임을 명시적으로 멈추거나 재개하는 일시정지 매니저 OptionMenuUi, GameOverUi

변경한 파일

Assets/Scripts/PauseManager.cs 추가

새로 만든 PauseManager는 싱글톤 방식으로 동작합니다. 씬에 이미 있으면 그 인스턴스를 사용하고, 없으면 런타임에 PauseManager 오브젝트를 만들어 사용합니다. 

public static void Pause()
{
    GetOrCreateInstance().SetPaused(true);
}

public static void Resume()
{
    GetOrCreateInstance().SetPaused(false);
}

내부에서는 Time.timeScale과 Time.fixedDeltaTime을 함께 조정합니다. timeScale만 바꾸면 물리 업데이트 쪽 설정이 이전 값과 어긋날 수 있으므로, 일시정지 상태에서는 fixedDeltaTime도 0으로 낮추고 재개 시 원래 기준값으로 복구합니다. 

private void SetPaused(bool isPaused)
{
    Time.timeScale = isPaused ? 0f : 1f;
    Time.fixedDeltaTime = isPaused ? 0f : _baseFixedDeltaTime;
}

Assets/Scripts/HitStopManager.cs 수정

SetGlobalTimeScale(float value) 메서드를 제거했습니다. 이제 HitStopManager는 Request(duration)와 Request(duration, stopScale)을 통해 히트스톱만 처리합니다.

Assets/Scripts/UI/OptionMenuUi.cs 수정

옵션 메뉴가 열릴 때는 PauseManager.Pause(), 닫힐 때는 PauseManager.Resume()을 호출하도록 변경했습니다. 

if (nextActive)
{
    PauseManager.Pause();
}
else
{
    PauseManager.Resume();
}

Assets/Scripts/UI/GameOverUi.cs 수정

게임오버 패널이 표시되면 PauseManager.Pause()로 게임을 멈추고, 타이틀로 돌아가기 전에는 PauseManager.Resume()으로 시간 배율을 복구합니다.

동작 흐름

공격 적중 시

  1. PlayerAttack이 공격 판정을 수행합니다.
  2. 실제로 데미지를 준 경우 HitStopManager.Request(...)를 호출합니다.
  3. HitStopManager가 짧은 시간 동안 Time.timeScale을 매우 낮춰 타격감을 만듭니다.
  4. 히트스톱 시간이 끝나면 이전 시간 배율로 복구합니다.

옵션 메뉴 열림/닫힘

  1. ESC 입력으로 옵션 패널을 켜거나 끕니다.
  2. 패널이 켜지면 PauseManager.Pause()를 호출해 게임 시간을 멈춥니다.
  3. 패널이 꺼지면 PauseManager.Resume()을 호출해 게임 시간을 재개합니다.

게임오버

  1. 플레이어 사망 이벤트 후 지정된 지연 시간이 지나면 게임오버 패널이 표시됩니다.
  2. 패널 표시 직후 PauseManager.Pause()로 게임 시간을 멈춥니다.
  3. 타이틀 버튼을 누르면 먼저 PauseManager.Resume()으로 시간을 복구한 뒤 씬을 이동합니다.

이번 수정의 핵심 학습 포인트

  • 클래스 이름과 책임이 일치해야 코드를 읽는 사람이 의도를 빠르게 이해할 수 있습니다.
  • 히트스톱은 전투 연출이고, 일시정지는 게임 상태 제어이므로 서로 다른 이유로 변경됩니다.
  • 서로 다른 이유로 변경되는 코드는 분리하는 편이 유지보수에 유리합니다.
  • Time.timeScale을 조정할 때는 Time.fixedDeltaTime도 함께 고려해야 합니다.

나중에 개선할 수 있는 점

현재 PauseManager는 단순히 Pause()와 Resume()으로 시간 배율을 0 또는 1로 바꿉니다. 프로젝트가 커지면 여러 시스템이 동시에 일시정지를 요청할 수 있습니다. 예를 들어 옵션 메뉴가 열린 상태에서 게임오버가 발생하면, 한쪽에서 Resume()을 호출했다고 해서 게임이 바로 풀리면 안 됩니다.

그 단계가 오면 아래처럼 요청자를 등록하고 해제하는 방식으로 확장할 수 있습니다. 

PauseManager.RequestPause(this);
PauseManager.ReleasePause(this);

이 방식은 하나 이상의 일시정지 요청이 남아 있으면 계속 멈춘 상태를 유지할 수 있어 더 안전합니다.

블로그 정리용 문장

이번 리팩터링의 목적은 HitStopManager에 섞여 있던 두 책임을 분리하는 것이었습니다. 히트스톱은 타격감을 위한 전투 연출이고, 옵션 메뉴나 게임오버의 정지는 게임 상태를 제어하는 일시정지입니다. 겉으로는 둘 다 Time.timeScale을 조절한다는 공통점이 있지만, 변경되는 이유와 사용되는 상황이 다르기 때문에 별도의 매니저로 나누는 편이 더 명확했습니다.

결과적으로 HitStopManager는 공격 적중 시 짧은 정지 연출만 담당하게 되었고, PauseManager는 옵션 메뉴와 게임오버처럼 게임을 멈추고 재개하는 흐름을 담당하게 되었습니다. 이 분리 덕분에 코드의 의도가 더 분명해졌고, 나중에 일시정지 규칙이 복잡해져도 전투 연출 코드에 영향을 덜 주는 구조가 되었습니다.
