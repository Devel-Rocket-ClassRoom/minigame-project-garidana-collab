# Project Origin 1주차 빌드 노트

## 빌드 개요

- 프로젝트명: ka_origin / 김경일의 모험: Origin
- 빌드 구분: 1주차 빌드
- 기준일: 2026-05-22
- 개발 목표: 플레이어 이동, 전투, 몬스터 전투, HUD, 경험치/레벨업 흐름을 연결해 1차 게임 루프를 확인할 수 있는 상태까지 구현
- 핵심 플레이 루프: 이동 -> 몬스터 조우 -> 공격/피격 -> 몬스터 사망 -> 경험치/골드 획득 -> HUD 반영

## 키 바인딩

- 방향키 - 이동
- 스페이스 - 공격
- 쉬프트 - 대쉬
- Esc - 게임 Pause 메뉴

## 주요 구현 내용

### 플레이어 조작 / 카메라

- 방향키 기반 8방향 이동을 구현.
- 쿼터뷰 카메라 추적 구조를 세팅.
- Shift 대시와 쿨다운 기반 대시 흐름을 구현.
- 스태미나 시스템을 추가해 대시 사용 시 스태미나가 감소하고, 일정 시간 이후 회복되도록 구성.
- 스태미나 고갈 시 대시 불가

### 플레이어 전투

- Space 입력 기반 기본 공격을 구현.
- 서로 다른 모션을 사용하는 3단 콤보 공격 흐름을 구현.
- 공격 콤보별로 데미지가 달라지도록 구성.
- 히트박스/공격 판정 흐름을 연결해 플레이어 공격이 몬스터 피격으로 연결.

### 몬스터 / 전투 시스템

- BaseMonster 기반의 몬스터 공통 전투 구조를 구현.
- NavMeshAgent 기반 몬스터 AI 흐름을 구성.
- 몬스터가 플레이어를 감지하고, 추적하고, 공격 범위에서 공격하도록 연결.
- 몬스터 피격, 사망, 경험치/골드 보상 지급 흐름을 연결했습니다.
- Chick, Egglet, Fungi 등 1주차 첫번째 구역 몬스터 프리팹을 구현.
- 몬스터 체력바 표시와 전투 중 HP 갱신 흐름을 구성.

### 플레이어 상태 / HUD

- 플레이어 HP, 스태미나, 경험치, 레벨, 골드 표시 HUD를 구현.
- PlayerHud.cs 를 통해 플레이어 상태 값이 UI에 실시간 반영되도록 구성.
- 레벨 시스템과 경험치 누적 구조를 1차 구현 (레벨 증가 효과 미구현).
- 플레이어 사망 시 게임오버 처리 흐름을 추가.

### UI / 메뉴

- 게임 종료 메뉴를 추가했습니다.
- 빌드 테스트용 기본 옵션/종료 UI 흐름을 구성.

### 씬 / 플레이 테스트

- 시작 마을 구역에 야생동물/몬스터 배치 및 전투 확인을 진행.
- 이동 -> 전투 -> 사망 -> 보상 (골드 및 경험지) 획득 흐름을 1차 검증.
- 몬스터 AI 배회/감지/공격 및 NavMesh 이동 흐름을 확인.




## 알려진 리스크 / 확인 필요

- 원래 목표 이슈였던 대시 쿨타임 UI 미구현, 기능은 동작
