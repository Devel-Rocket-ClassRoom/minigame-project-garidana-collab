요약

전투 중 플레이어가 하늘로 솟는 현상은 넉백 코드나 데미지 처리 코드보다는, 동적 Rigidbody 플레이어와 NavMeshAgent로 이동하는 non-trigger 몬스터 Collider가 겹칠 때 Unity 물리 엔진이 겹침을 위쪽으로 보정하면서 발생하는 문제로 판단됩니다.

방패 Collider를 isTrigger로 바꿔도 문제가 남는 이유는 플레이어의 본체 CapsuleCollider와 몬스터의 SphereCollider가 여전히 실제 물리 충돌을 하기 때문입니다.

문제가 생기는 흐름

  1. 몬스터는 BaseMonster.ChasePlayer()에서 agent.SetDestination(player.position)으로 플레이어를 추격합니다.
  2. 플레이어는 PlayerMovement.Move()에서 Rigidbody.MovePosition으로 움직입니다.
  3. 몬스터의 SphereCollider와 플레이어의 CapsuleCollider가 전투 중 겹칩니다.
  4. 몬스터는 Rigidbody 없이 NavMeshAgent가 직접 Transform을 움직이는 구조라 물리 엔진 입장에서는 충돌 해석이 불안정해질 수 있습니다.
  5. 겹침을 해소하는 방향이 지형 경사나 Collider 접촉면 때문에 위쪽으로 잡히면, 플레이어 Rigidbody가 위로 튀어오릅니다.
핵심 원인: 충돌 자체가 문제가 아니라, NavMeshAgent가 움직이는 non-trigger 몬스터 Collider와 동적 Rigidbody 플레이어가 겹친 상태를 물리 엔진이 강제로 풀면서 위쪽 보정이 발생하는 것이 문제입니다.

이미 확인하거나 시도한 내용

항목 결과 판단
방패 Collider를 Trigger로 변경 문제가 계속 발생함. 방패만 원인은 아님. 본체 CapsuleCollider와 몬스터 SphereCollider의 충돌이 남아 있음.
Damageable과 플레이어 레이어 충돌 비활성화 튀어오름은 줄일 수 있지만 플레이어와 몬스터가 서로 통과함. 사용자 의도와 맞지 않음.
공격/데미지 코드 확인 AddForce, 위쪽 넉백, 강제 점프 코드 없음. 코드로 위로 힘을 주는 문제는 아님.
Physics Material 안내 Bounciness=0, 마찰 최소 재질을 만들도록 안내함. 보조 해결책. 근본 해결은 Rigidbody/NavMeshAgent 충돌 구조 보정.

권장 해결 방향

1. 몬스터에 Kinematic Rigidbody 추가

플레이어와 몬스터의 충돌은 유지하되, 몬스터 Collider가 물리 시스템에 더 일관되게 참여하도록 몬스터 프리팹에 Rigidbody를 추가합니다.

  • Rigidbody 추가
  • Is Kinematic = true
  • Use Gravity = false
  • Collision Detection = Continuous Speculative
  • 몬스터 SphereCollider는 non-trigger 유지

2. 플레이어의 위쪽 속도 제한

현재 게임 구조상 점프가 없다면, 충돌 보정으로 생기는 양수 Y 속도를 제거하는 방어 코드를 넣는 것이 효과적입니다.

private void FixedUpdate()
{
    if (_playerStats != null && _playerStats.IsDead)
    {
        return;
    }

    Move();

    Vector3 velocity = _playerRigidbody.linearVelocity;
    if (velocity.y > 0f)
    {
        velocity.y = 0f;
        _playerRigidbody.linearVelocity = velocity;
    }
}

주의: 추후 점프, 넉업, 점프 패드 같은 기능이 추가되면 이 방식은 조건부로 바꿔야 합니다.

3. 플레이어 Rigidbody 설정 보강

  • Collision Detection: Discrete에서 Continuous Dynamic으로 변경
  • Interpolation: Interpolate로 변경
  • Constraints: 현재처럼 Freeze Rotation 유지

4. Physics Material 적용

플레이어와 몬스터 Collider에 튕김 없는 Physics Material을 적용합니다.

Dynamic Friction: 0
Static Friction: 0
Bounciness: 0
Friction Combine: Minimum
Bounce Combine: Minimum

적용 대상은 플레이어 CapsuleCollider와 몬스터 SphereCollider입니다. 방패 Collider가 물리 충돌 목적이 아니라면 Trigger로 두는 것이 안전합니다.

5. 지형 Collider 경고 수정

SM_Env_Road_Corner_01의 negative scale BoxCollider 경고는 별도로 수정해야 합니다. 지형 충돌 형상이 예상과 달라질 수 있어, 전투 중 튀어오름 현상을 더 쉽게 유발할 수 있습니다.

권장 우선순위

  1. 우선 몬스터 프리팹에 Kinematic Rigidbody 추가
  2. 우선 플레이어 Rigidbody의 위쪽 속도 제한 코드 추가
  3. 다음 플레이어 Rigidbody를 Continuous Dynamic, Interpolate로 변경
  4. 다음 No Bounce Physics Material 적용
  5. 정리 negative scale이 있는 지형 Collider 수정
최종 방향: 플레이어와 몬스터의 물리 충돌은 유지하되, 몬스터를 Kinematic Rigidbody 기반 충돌체로 만들고 플레이어의 비정상 Y 속도를 제한하면, 서로 막히는 감각을 유지하면서 하늘로 솟는 현상을 줄일 수 있습니다.
