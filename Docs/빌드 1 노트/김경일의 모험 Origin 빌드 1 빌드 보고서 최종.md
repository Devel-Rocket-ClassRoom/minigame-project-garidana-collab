## 2. 빌드 목표

- 목표 1: 기본 플레이어 조작 구현
  - 방향키 8방향 이동, Shift 대시, Space 기본 공격, 3단 콤보 공격까지 1차 플레이 조작 흐름을 완성.
- 목표 2: 테스트용 시작 구역 구성 및 몬스터 배치
  - 시작 마을/시작 구역에 테스트용 몬스터를 배치하고, 플레이어가 이동하며 전투를 진행할 수 있도록 구성.
- 목표 3: 플레이어와 몬스터 전투 루프 구현
  - 몬스터 감지, 추적, 공격, 피격, 사망, 경험치/골드 보상 지급, HUD 반영까지 이어지는 1주차 핵심 루프를 구현.

## 6. 개선 사항

- 대시 쿨타임 UI
- 경험치 바 위치 및 HUD 가독성 개선
- 빌드 실행 파일 직접 실행 검증 결과 기록

- 몬스터에세 플레이어가 밀림 이슈

## 7. 빌드 플레이 소감

- 조작감: 이동, 대시, 공격 입력의 기본 구조가 갖춰져 1차 액션 RPG 조작 흐름은 성립한다.
- 전투감: 3단 콤보와 몬스터 피격/사망, 보상 지급까지 연결되어 전투 루프의 기본 형태가 확인된다.
- UI 가독성: HP, 스태미나, 경험치, 레벨, 골드 HUD는 구현되었으나 경험치 바 위치 조정이 필요해 보인다.
- 난이도: 중간. 1주차 테스트 기준으로 몬스터와 전투 루프를 확인하기 적당한 수준이다.
- 전체 흐름: 플레이어 이동 -> 몬스터 조우 -> 전투 -> 처치 보상 -> HUD 반영의 핵심 흐름이 자연스럽게 이어진다.

## 8. 다음 빌드 개선 계획

- 대시 쿨타임 UI를 추가해 대시 사용 가능 여부를 명확히 표시한다.
- HUD 레이아웃을 정리해 경험치 바와 상태 정보의 가독성을 높인다.
- 황무지 구역 레이아웃과 라이팅을 추가해 2번째 구역 진입 흐름을 만든다.
- F키 상호작용 기본 구조를 구현해 상자/NPC 상호작용 확장 기반을 만든다.
- 몬스터 드롭 아이템, 퀘스트 아이템, 퀘스트 HUD를 다음 빌드 범위에서 연결한다.
