# Project Origin 2주차 빌드 목표

## 빌드 개요

- 프로젝트명: ka_origin / 김경일의 모험: Origin
- 빌드 구분: 2주차 빌드
- 기간: 2026-05-26 ~ 2026-06-01
- Notion 기준 목표: 구역 사이클 구현 (2차 빌드)
- 핵심 플레이 루프: 퀘스트 진행 -> 레벨 업 -> 공격 강화 -> 다음 구역 진입

## 2주차 핵심 목표

2주차 빌드는 1주차에 만든 기본 전투 루프를 확장해서, 플레이어가 퀘스트와 아이템을 통해 다음 구역으로 진행할 수 있는 구조를 만드는 것이 목표. 한구역의 퀘스트를 완료하고 10레벨을 달성하여 공격 강화까지 구현.
- 퀘스트 시스템 구현 (퀘스트 수락, 완료, 공격강화)
- 장비창 시스템 구현
- 어둠 숲에서 오크 주둔지 까지 완성 (몬스터, 퀘스트까지, 어둠 숲 엘리트 몬스터 한마리까지)


## 주요 구현 범위

### 퀘스트 진행 구조

- NPC 또는 상호작용 오브젝트를 통해 퀘스트를 수락할 수 있도록 구성합니다.
- 퀘스트 진행 조건은 몬스터 처치, 특정 아이템 수집, 퀘스트 아이템 제출을 기준으로 잡습니다.
- 퀘스트 완료 시 보상 지급과 기본 공격 강화 및 다음 구역 진행 조건 해금을 연결합니다.
- 퀘스트는 한 번에 하나씩 진행하는 구조를 기본으로 합니다.

### 인벤토리 (장비창) 구현

- 아이템 획득 후 보유할 수 있는 기본 인벤토리 UI를 구현합니다.
- 보유 장비 열람 및 탈착
- 아이템 드래그 앤 드롭 이동을 구현합니다.


### 아이템 획득 / 장비 획득 / 장착

- 몬스터 처치 (퀘스트 아이템) 또는 상자 상호작용(무기 혹은 방패)을 통해 아이템을 획득할 수 있도록 구성합니다.
- 무기와 방패 등 장비 아이템을 인벤토리에 추가할 수 있도록 합니다.
- 장비 장착 시 공격력, 체력 등 플레이어 능력치 변화가 가능하도록 구조를 설계합니다.
- 이후 캐릭터 외형 교체까지 확장할 수 있도록 장비 데이터와 장착 슬롯을 분리합니다.

### 체력 회복 사용

- 체력 회복 시 플레이어 HP 회복 등의 효과를 적용합니다.
- 사용 후 사용 횟수이 감소하도록 처리합니다.


### 퀘스트 아이템 소모

- 퀘스트 완료 조건에 필요한 아이템을 인벤토리에서 확인합니다.
- 퀘스트 완료 시 필요한 퀘스트 아이템을 소모합니다.
- 퀘스트 아이템과 일반 아이템을 구분할 수 있는 데이터 구조를 만듭니다.
- 퀘스트 완료 후 다음 구역 또는 다음 퀘스트가 열리도록 연결합니다.

### 몬스터 아이템 드랍 구현

- 몬스터 처치 시 드랍 테이블에 따라 아이템이 생성되도록 구현합니다.
- 구역별 퀘스트 아이템 드랍을 연결합니다.
- 경험치와 골드는 기존 보상 흐름을 유지하되, 필요하다면 드랍 이펙트 또는 획득 연출을 추가합니다.
- 드랍 아이템은 플레이어가 가까이 가면 자동 획득

### 경험치 / 골드 드랍 이펙트

- 몬스터 처치 시 경험치와 골드 획득이 시각적으로 드러나도록 합니다.
- 획득 이펙트, 텍스트 팝업, HUD 갱신 중 최소 1개 이상을 명확히 보여줍니다.
- 플레이어가 보상을 받았다는 피드백을 강화합니다.

### 공격 강화 이펙트

- 레벨 10 단위로 기본 공격이 강화되는 구조를 구현합니다.
- 강화 단계에 따라 데미지, 공격 범위, 이펙트 색상 또는 연출이 달라지도록 설계합니다.
- 2주차에서는 최소 1단계 강화가 실제 플레이에서 확인되도록 만드는 것을 우선 목표로 합니다.
- Notion 기준 구조는 `Normal -> Enhanced 1 -> Enhanced 2 -> Enhanced 3` 방향입니다.

### 구역 및 몬스터 추가

- 시작 구역 이후의 다음 구역을 추가합니다.
- 황무지/사막 구역을 우선 후보로 두고, POLYGON Adventure/Western Pack 기반 레이아웃을 구성합니다.
- 새 구역에 맞는 몬스터를 추가하거나 기존 몬스터의 스탯을 조정합니다.
- 구역별 몬스터 스탯 스케일링 구조를 준비합니다.
- 다음 구역 진입 조건은 퀘스트 완료 또는 레벨/공격 강화 달성과 연결합니다.

## 2주차 완료 기준

| 구분 | 완료 기준 |
| --- | --- |
| 퀘스트 | 퀘스트 수락, 진행, 완료, 보상 지급 흐름이 1개 이상 동작 |
| 인벤토리 | 아이템 획득 후 인벤토리에 표시되고 이동/사용/장착 중 핵심 기능이 동작 |
| 아이템 드랍 | 몬스터 처치 시 퀘스트 아이템 또는 보상 아이템이 드랍됨 |
| 소모품 | 체력 포션 등 소모품을 사용할 수 있고 수량이 감소함 |
| 공격 강화 | 레벨 또는 진행 조건에 따라 기본 공격 강화가 1단계 이상 적용됨 |
| 구역 확장 | 시작 구역 이후 다음 구역으로 이동하거나 진입 조건을 확인할 수 있음 |
| 몬스터 확장 | 추가 구역용 몬스터 또는 스탯 스케일링이 적용됨 |
| HUD/UI | 퀘스트, 인벤토리, 보상, 플레이어 상태가 플레이 중 확인 가능 |

## 권장 작업 순서

1. 인벤토리 데이터 구조와 UI 기본 틀 구현
2. 아이템 데이터와 드랍 아이템 프리팹 구조 구현
3. 몬스터 드랍 테이블과 퀘스트 아이템 획득 연결
4. 퀘스트 수락/진행/완료 구조 구현
5. 소모품 사용과 장비 장착 구조 구현
6. 레벨 10 기준 기본 공격 강화 구조 구현
7. 황무지 구역과 구역별 몬스터/스탯 확장
8. 2차 빌드 플레이테스트 및 버그 수정

## 우선순위

| 우선순위 | 작업 |
| --- | --- |
| 1 | 퀘스트 진행 -> 완료 -> 보상 지급 흐름 |
| 2 | 인벤토리와 아이템 획득/소모 구조 |
| 3 | 몬스터 아이템 드랍 및 퀘스트 아이템 연결 |
| 4 | 기본 공격 강화 시스템 |
| 5 | 다음 구역 및 몬스터 추가 |
| 6 | 이펙트와 UI 피드백 폴리싱 |

## 리스크 / 확인 필요

- 인벤토리와 장비 시스템은 공수가 큰 편이므로, 처음부터 완전한 장비 외형 교체까지 욕심내면 일정이 밀릴 수 있습니다.
- 퀘스트 시스템은 NPC 대화, 인벤토리, 아이템 소모, 보상 지급과 연결되므로 의존성이 많습니다.
- 공격 강화는 레벨 시스템과 전투 밸런스에 연결되므로, 먼저 데이터 구조를 단순하게 잡고 이펙트는 최소 버전부터 붙이는 편이 안전합니다.
- 구역 확장은 NavMesh, 라이팅, 몬스터 스폰, 경계 처리까지 같이 흔들리므로 맵 배치를 먼저 고정한 뒤 기능을 붙이는 순서가 좋습니다.
- 2주차 빌드 목표는 범위가 넓기 때문에, 최소 완성 기준은 `퀘스트 1개 + 퀘스트 아이템 드랍 + 인벤토리 획득/소모 + 다음 구역 진입`으로 잡는 것이 현실적입니다.

## 2주차 빌드 요약

2주차 빌드는 전투만 가능한 상태에서 한 단계 나아가, 플레이어가 퀘스트를 받고, 몬스터를 잡아 아이템을 얻고, 인벤토리에서 아이템을 관리하며, 보상과 공격 강화를 통해 다음 구역으로 진행하는 구조를 만드는 빌드입니다. 핵심은 모든 시스템을 완벽하게 만드는 것이 아니라, `퀘스트 진행 -> 아이템 획득 -> 퀘스트 완료 -> 성장 -> 다음 구역`의 한 사이클을 실제 플레이에서 확인 가능하게 만드는 것입니다.
