전체 구조 & 전제 조건

🎯 이번 구현 범위

상자(ChestInteractable)를 열면 검 또는 방패 ItemData가 PlayerInventory에 추가됩니다. 인벤토리 UI(TAB키)에서 슬롯을 클릭하면 EquipmentManager가 모델을 교체하고 PlayerStats 스탯을 적용합니다. 방어구(Helmet/Chest/Legs)는 이번 범위에서 제외합니다.

✅ 이미 있는 것 (건드리지 않음)

ItemData.cs — 완성 (equipObjectName, attackBonus 포함)
PlayerStats.cs — IncreaseAttackPower(), IncreaseMaxHealth() 존재
DropEntry.cs — 드랍 확률 구조체 완성
ChestInteractable.cs — 뚜껑 애니메이션 완성, 드랍 로직만 없음
Sword/Shield .asset 30종 전부 완성 (equipObjectName 세팅 완료)

이 문서는 코드를 직접 수정하지 않고 Cowork에게 작업을 지시하기 위한 명세서입니다. 각 단계를 순서대로 Cowork에 전달하세요. 수정이 필요한 파일과 신규 생성할 파일이 명확히 구분되어 있습니다.

SCOPE

Sword1.asset의 equipObjectName은 Sword_1, Shield1.asset은 shield_1입니다. EquipmentManager는 이 문자열을 기준으로 캐릭터 모델의 자식 오브젝트를 찾아 활성화/비활성화합니다. 따라서 캐릭터 프리팹의 본(bone) 구조 안에 Sword_1, Sword_2… shield_1… 이름의 오브젝트가 미리 있어야 합니다. 없다면 5단계 씬 연결에서 직접 배치합니다.

01

PlayerInventory.cs 신규 생성

STEP 01 인벤토리 데이터 컴포넌트 예상 30분

플레이어 GameObject에 붙는 데이터 전용 컴포넌트입니다. UI나 모델 교체 로직은 여기에 없습니다. 아이템 목록을 들고 있고, 추가/제거/조회 메서드만 제공합니다.

생성 경로 NEW
Assets/Scripts/Player/PlayerInventory.cs 
using System;
using System.Collections.Generic;
using UnityEngine;

public class PlayerInventory : MonoBehaviour
{
    // ── 인스펙터 설정 ───────────────────────────────────
    [SerializeField]
    [Range(1, 40)]
    private int _maxSlots = 20;

    // ── 이벤트 ───────────────────────────────────────────
    // InventoryUi가 구독해서 슬롯을 다시 그립니다.
    public event Action InventoryChanged;

    // ── 내부 상태 ─────────────────────────────────────────
    private readonly List<ItemData> _items = new List<ItemData>();

    // ── 공개 조회 ─────────────────────────────────────────
    public IReadOnlyList<ItemData> Items => _items;
    public bool IsFull => _items.Count >= _maxSlots;
    public int Count => _items.Count;

    // ── AddItem ───────────────────────────────────────────
    // 성공하면 true, 인벤토리가 꽉 찼으면 false 반환
    public bool AddItem(ItemData data)
    {
        if (data == null || IsFull)
        {
            Debug.Log($"[Inventory] 추가 실패: data={data?.displayName}, full={IsFull}");
            return false;
        }

        _items.Add(data);
        InventoryChanged?.Invoke();

        Debug.Log($"[Inventory] 추가: {data.displayName} ({_items.Count}/{_maxSlots})");
        return true;
    }

    // ── RemoveItem ────────────────────────────────────────
    // 장착 시 또는 버릴 때 호출
    public bool RemoveItem(ItemData data)
    {
        bool removed = _items.Remove(data);
        if (removed)
        {
            InventoryChanged?.Invoke();
            Debug.Log($"[Inventory] 제거: {data.displayName}");
        }
        return removed;
    }

    // ── HasItem ───────────────────────────────────────────
    public bool HasItem(ItemData data) => _items.Contains(data);
}
CHECK

이 스크립트는 데이터만 들고 있습니다. UI 갱신은 InventoryChanged 이벤트를 InventoryUi가 구독하는 방식으로 처리합니다. MonoBehaviour를 상속하기 때문에 플레이어 프리팹에 컴포넌트로 추가해야 합니다.

02

EquipmentManager.cs 신규 생성

STEP 02 장착 & 모델 교체 컴포넌트 예상 1~1.5시간

장착 슬롯(Sword 1개, Shield 1개)을 관리하고 ItemData.equipObjectName을 기반으로 캐릭터 모델 오브젝트를 교체합니다. 스탯 변동은 이전 장착 아이템의 보너스를 빼고 새 아이템의 보너스를 더하는 방식으로 처리합니다.

CRITICAL

스탯 중복 방지가 핵심입니다. PlayerStats.IncreaseAttackPower()는 더하기만 합니다. 따라서 장착 교체 시 이전 아이템의 보너스를 음수로 먼저 적용해서 뺀 뒤, 새 아이템 보너스를 더해야 합니다. _equippedSword, _equippedShield 필드에 현재 장착 정보를 항상 보관하는 이유입니다.

생성 경로 NEW
Assets/Scripts/Player/EquipmentManager.cs 
using UnityEngine;
using System;

public class EquipmentManager : MonoBehaviour
{
    // ── 인스펙터 설정 ───────────────────────────────────
    [Header("References")]
    [SerializeField] private PlayerStats      _playerStats;
    [SerializeField] private PlayerInventory  _inventory;

    [Header("Weapon Mount Points")]
    // 캐릭터 Armature 안의 오른손 본 또는 무기 마운트 부모 Transform
    [SerializeField] private Transform _swordMountParent;
    // 왼손 본 또는 방패 마운트 부모 Transform
    [SerializeField] private Transform _shieldMountParent;

    // ── 이벤트 ───────────────────────────────────────────
    public event Action EquipmentChanged;

    // ── 현재 장착 아이템 ─────────────────────────────────
    private ItemData _equippedSword;
    private ItemData _equippedShield;

    public  ItemData  EquippedSword  => _equippedSword;
    public  ItemData  EquippedShield => _equippedShield;

    // ── 초기화 ───────────────────────────────────────────
    private void Awake()
    {
        if (_playerStats == null)
            _playerStats = GetComponent<PlayerStats>();
        if (_inventory == null)
            _inventory = GetComponent<PlayerInventory>();
    }

    // ── EquipItem (외부에서 호출) ─────────────────────────
    // InventoryUi의 슬롯 클릭 이벤트에서 호출합니다.
    public void EquipItem(ItemData data)
    {
        if (data == null || !data.IsEquipment)
        {
            Debug.LogWarning("[Equipment] 장착 불가: 장비 아이템이 아닙니다.");
            return;
        }

        switch (data.equipmentPart)
        {
            case EquipmentPart.Sword:
                SwapEquip(ref _equippedSword, data, _swordMountParent);
                break;
            case EquipmentPart.Shield:
                SwapEquip(ref _equippedShield, data, _shieldMountParent);
                break;
            default:
                Debug.LogWarning($"[Equipment] {data.displayName}: 검/방패만 지원합니다.");
                break;
        }
    }

    // ── UnequipItem (해제) ───────────────────────────────
    public void UnequipItem(EquipmentPart part)
    {
        switch (part)
        {
            case EquipmentPart.Sword:
                if (_equippedSword != null)
                {
                    RemoveStats(_equippedSword);
                    HideEquipModel(_equippedSword.equipObjectName, _swordMountParent);
                    _equippedSword = null;
                    EquipmentChanged?.Invoke();
                }
                break;
            case EquipmentPart.Shield:
                if (_equippedShield != null)
                {
                    RemoveStats(_equippedShield);
                    HideEquipModel(_equippedShield.equipObjectName, _shieldMountParent);
                    _equippedShield = null;
                    EquipmentChanged?.Invoke();
                }
                break;
        }
    }

    // ── IsEquipped ───────────────────────────────────────
    // InventoryUi에서 슬롯 하이라이트 여부 판단에 사용
    public bool IsEquipped(ItemData data)
    {
        return data != null &&
               (data == _equippedSword || data == _equippedShield);
    }

    // ══════════════════════════════════════════════════════
    // Private helpers
    // ══════════════════════════════════════════════════════

    private void SwapEquip(ref ItemData slot, ItemData newItem, Transform mountParent)
    {
        // 1. 이전 장착 해제 (스탯 제거 → 모델 숨기기)
        if (slot != null)
        {
            RemoveStats(slot);
            HideEquipModel(slot.equipObjectName, mountParent);
        }

        // 2. 새 아이템 장착 (스탯 적용 → 모델 보이기)
        slot = newItem;
        ApplyStats(slot);
        ShowEquipModel(slot.equipObjectName, mountParent);

        EquipmentChanged?.Invoke();

        Debug.Log($"[Equipment] 장착: {newItem.displayName}");
    }

    // 스탯 적용
    private void ApplyStats(ItemData data)
    {
        if (_playerStats == null || data == null) return;
        if (data.attackBonus    != 0) _playerStats.IncreaseAttackPower(data.attackBonus);
        if (data.maxHealthBonus != 0) _playerStats.IncreaseMaxHealth(data.maxHealthBonus);
    }

    // 스탯 제거 (음수로 넘김)
    private void RemoveStats(ItemData data)
    {
        if (_playerStats == null || data == null) return;
        if (data.attackBonus    != 0) _playerStats.IncreaseAttackPower(-data.attackBonus);
        if (data.maxHealthBonus != 0) _playerStats.IncreaseMaxHealth(-data.maxHealthBonus);
    }

    // mountParent 하위에서 equipObjectName과 일치하는 오브젝트 활성화
    private void ShowEquipModel(string equipName, Transform mountParent)
    {
        if (mountParent == null || string.IsNullOrEmpty(equipName)) return;

        Transform target = mountParent.Find(equipName);
        if (target != null)
        {
            target.gameObject.SetActive(true);
        }
        else
        {
            Debug.LogWarning($"[Equipment] '{equipName}' 오브젝트를 '{mountParent.name}' 하위에서 찾을 수 없습니다.");
        }
    }

    // mountParent 하위에서 모든 자식 비활성화 후 해당 이름만 active
    private void HideEquipModel(string equipName, Transform mountParent)
    {
        if (mountParent == null || string.IsNullOrEmpty(equipName)) return;

        Transform target = mountParent.Find(equipName);
        if (target != null)
        {
            target.gameObject.SetActive(false);
        }
    }
}

인스펙터 연결 항목

필드연결 대상비고
_playerStatsPlayer GameObject의 PlayerStats 컴포넌트Awake에서 자동 찾기
_inventoryPlayer GameObject의 PlayerInventory 컴포넌트Awake에서 자동 찾기
_swordMountParent캐릭터 Armature 안의 오른손 본 또는 무기 마운트 Transform직접 드래그 필수
_shieldMountParent캐릭터 Armature 안의 왼손 본 또는 방패 마운트 Transform직접 드래그 필수
ABOUT

ShowEquipModel은 mountParent.Find(equipObjectName)으로 자식 오브젝트를 찾습니다. Sword1.asset의 equipObjectName이 Sword_1이므로, _swordMountParent 하위에 Sword_1이라는 이름의 오브젝트가 있어야 합니다. 해당 오브젝트가 없으면 Cowork에 "캐릭터 프리팹의 swordMountParent 하위에 Sword_1, Sword_2, Sword_3, Sword_4, Sword_5, Sword_6 이름의 자식 오브젝트를 확인해 달라"고 요청하세요.

ChestInteractable.cs 수정

STEP 03 상자 드랍 로직 추가 예상 20~30분

기존 파일에서 주석 처리된 드랍 관련 코드는 전부 삭제하고, 아래 내용으로 교체합니다. 뚜껑 애니메이션(RotateLid, RotateLid Coroutine)은 그대로 유지합니다.

수정 경로 MODIFY

추가할 필드 (클래스 상단)

[Header("Drop Settings")]
// DropEntry는 이미 존재: ItemData item + float dropChance (0~1)
[SerializeField] private DropEntry[] _dropTable;

// 플레이어 프리팹에서 자동 탐색 (Awake), 또는 인스펙터에서 드래그
[SerializeField] private PlayerInventory _playerInventory;

Awake 메서드 추가 (또는 기존 Awake에 병합)

private void Awake()
{
    // 씬에 플레이어가 하나뿐이라는 가정
    if (_playerInventory == null)
        _playerInventory = FindFirstObjectByType<PlayerInventory>();

Interact() 메서드 수정

기존 Interact()에서 OpenLid()만 호출하던 부분을 아래와 같이 교체합니다.

public void Interact(GameObject interactor)
{
    OpenLid();      // 기존 애니메이션 유지
    RollDrops();   // 드랍 로직 추가
}

private void RollDrops()
{
    if (_dropTable == null || _dropTable.Length == 0) return;
    if (_playerInventory == null)
    {
        Debug.LogWarning("[Chest] PlayerInventory를 찾을 수 없습니다.");
        return;
    }

    foreach (DropEntry entry in _dropTable)
    {
        if (entry.item == null) continue;
        if (Random.value <= entry.dropChance)
        {
            bool added = _playerInventory.AddItem(entry.item);
            Debug.Log($"[Chest] 드랍: {entry.item.displayName}, 인벤 추가={added}");
        }
    }
}

인스펙터에서 DropTable 설정 방법

씬에 배치된 각 Chest 프리팹의 Inspector에서 Drop Table의 Size를 원하는 수만큼 설정한 뒤, 각 Entry의 Item에 Assets/Scripts/ScriptableObjects/Items/Swords/Sword2.asset(또는 Shield2~6.asset)를 드래그합니다. Drop Chance는 0~1 사이 값으로 설정합니다 (0.5 = 50%).

예시 상자ItemDrop Chance
황무지 상자Sword2.asset0.5
황무지 상자Shield2.asset0.4
어둠 숲 상자Sword3.asset0.45
오크 상자Sword5.asset0.35
NOTE

Sword1.asset과 Shield1.asset은 isDefaultEquipment = true이고 attackBonus = 0입니다. 기본 장착 아이템이므로 드랍 테이블에 넣지 않아도 됩니다. Sword2 이상부터 드랍되도록 설계하는 것을 권장합니다.

04

InventoryUi.cs 비어있음 → 구현

STEP 04 인벤토리 UI 코드 구현 예상 1~1.5시간

현재 파일이 빈 껍데기입니다. 전체를 아래 코드로 교체합니다. TAB키로 패널을 열고 닫으며, 슬롯 클릭 시 즉시 장착합니다. 드래그앤드롭은 이번 단계에서 제외하고 기본 동작 확인 후 나중에 추가합니다.

수정 경로 EMPTY → IMPLEMENT
Assets/Scripts/UI/InventoryUi.cs 
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.UI;
using TMPro;

public class InventoryUi : MonoBehaviour
{
    // ── 인스펙터 연결 ────────────────────────────────────
    [Header("References")]
    [SerializeField] private PlayerInventory  _inventory;
    [SerializeField] private EquipmentManager _equipment;

    [Header("UI")]
    // 인벤토리 전체 패널 루트 오브젝트 (SetActive 토글 대상)
    [SerializeField] private GameObject       _panelRoot;
    // 슬롯 버튼들이 들어갈 Grid Layout Group 부모 Transform
    [SerializeField] private Transform          _slotContainer;
    // 슬롯 하나의 프리팹 (Button + Image + TextMeshProUGUI 구성)
    [SerializeField] private InventorySlotUi    _slotPrefab;

    [Header("장착 슬롯 표시 (선택)")]
    [SerializeField] private Image  _equippedSwordIcon;
    [SerializeField] private Image  _equippedShieldIcon;

    // ── 내부 상태 ─────────────────────────────────────────
    private readonly List<InventorySlotUi> _slots = new List<InventorySlotUi>();
    private bool _isOpen = false;

    // ── 초기화 ───────────────────────────────────────────
    private void Awake()
    {
        if (_inventory == null)
            _inventory = FindFirstObjectByType<PlayerInventory>();
        if (_equipment == null)
            _equipment = FindFirstObjectByType<EquipmentManager>();

        // 패널 닫힌 상태로 시작
        SetPanelActive(false);
    }

    private void OnEnable()
    {
        if (_inventory != null)
            _inventory.InventoryChanged += RefreshSlots;
        if (_equipment != null)
            _equipment.EquipmentChanged += RefreshSlots;
    }

    private void OnDisable()
    {
        if (_inventory != null)
            _inventory.InventoryChanged -= RefreshSlots;
        if (_equipment != null)
            _equipment.EquipmentChanged -= RefreshSlots;
    }

    // ── TAB 키 처리 ──────────────────────────────────────
    // PlayerInputReader.cs에 OnInventory(InputValue) 콜백 추가 후
    // InputSystem에서 TAB 키를 "Inventory" 액션으로 바인딩하면
    // 아래 메서드를 직접 호출합니다. 그 전까지는 Update에서 임시 처리.
    private void Update()
    {
        if (Input.GetKeyDown(KeyCode.Tab))
            TogglePanel();
    }

    public void TogglePanel()
    {
        SetPanelActive(!_isOpen);
    }

    private void SetPanelActive(bool active)
    {
        _isOpen = active;
        if (_panelRoot != null)
            _panelRoot.SetActive(active);

        if (active)
            RefreshSlots();
    }

    // ── 슬롯 갱신 ────────────────────────────────────────
    // 인벤토리 변경 또는 장착 변경 시 자동 호출됩니다.
    private void RefreshSlots()
    {
        if (_inventory == null || _slotContainer == null) return;

        // 슬롯 수 맞추기
        int itemCount = _inventory.Count;
        EnsureSlotCount(itemCount);

        // 슬롯 데이터 채우기
        for (int i = 0; i < _slots.Count; i++)
        {
            if (i < itemCount)
            {
                ItemData data = _inventory.Items[i];
                bool isEquipped = _equipment != null && _equipment.IsEquipped(data);
                _slots[i].Setup(data, isEquipped, OnSlotClicked);
                _slots[i].gameObject.SetActive(true);
            }
            else
            {
                _slots[i].gameObject.SetActive(false);
            }
        }

        // 장착 슬롯 아이콘 갱신 (있을 때만)
        RefreshEquippedIcons();
    }

    private void OnSlotClicked(ItemData data)
    {
        if (data == null || _equipment == null) return;
        _equipment.EquipItem(data);
        // EquipmentChanged 이벤트 → RefreshSlots 자동 호출
    }

    private void EnsureSlotCount(int needed)
    {
        while (_slots.Count < needed)
        {
            InventorySlotUi slot = Instantiate(_slotPrefab, _slotContainer);
            _slots.Add(slot);
        }
    }

    private void RefreshEquippedIcons()
    {
        if (_equippedSwordIcon != null)
        {
            ItemData sw = _equipment?.EquippedSword;
            _equippedSwordIcon.sprite  = sw?.icon;
            _equippedSwordIcon.enabled = sw != null;
        }
        if (_equippedShieldIcon != null)
        {
            ItemData sh = _equipment?.EquippedShield;
            _equippedShieldIcon.sprite  = sh?.icon;
            _equippedShieldIcon.enabled = sh != null;
        }
    }
}

같은 파일에 함께 작성 — InventorySlotUi.cs (별도 파일 또는 하단 추가)

슬롯 하나의 버튼 동작을 담당하는 작은 컴포넌트입니다. InventorySlotUi.cs라는 이름으로 별도 생성하거나, InventoryUi.cs 하단에 같이 넣어도 됩니다.

생성 경로 NEW
Assets/Scripts/UI/InventorySlotUi.cs 
using System;
using UnityEngine;
using UnityEngine.UI;
using TMPro;

public class InventorySlotUi : MonoBehaviour
{
    [SerializeField] private Image             _iconImage;
    [SerializeField] private TextMeshProUGUI   _nameText;
    [SerializeField] private Button            _button;
    // 장착 중일 때 보여줄 하이라이트 오브젝트 (테두리 이미지 등)
    [SerializeField] private GameObject       _equippedHighlight;

    private ItemData          _data;
    private Action<ItemData> _onClicked;

    public void Setup(ItemData data, bool isEquipped, Action<ItemData> onClicked)
    {
        _data      = data;
        _onClicked = onClicked;

        if (_iconImage != null)
        {
            _iconImage.sprite  = data.icon;
            _iconImage.enabled = data.icon != null;
        }
        if (_nameText != null)
            _nameText.text = data.displayName;

        if (_equippedHighlight != null)
            _equippedHighlight.SetActive(isEquipped);

        _button.onClick.RemoveAllListeners();
        _button.onClick.AddListener(HandleClick);
    }

    private void HandleClick()
    {
        _onClicked?.Invoke(_data);
    }
}

씬 & 프리팹 연결

STEP 05 Unity 에디터 작업 (코딩 없음) 예상 30~60분

5-1. 플레이어 프리팹에 컴포넌트 추가

Assets/Prefabs/Player/Player.prefab을 열고 루트 오브젝트에 아래 컴포넌트를 추가합니다.

  • Add Component → PlayerInventory (Max Slots = 20)
  • Add Component → EquipmentManager

5-2. 무기 마운트 포인트 확인 & 연결

캐릭터 Armature(Hierarchy에서 Player → Armature 하위)를 펼쳐 오른손 본을 찾습니다. 보통 hand_r, mixamorig:RightHand 등의 이름입니다.

  • 해당 본을 EquipmentManager의 Sword Mount Parent에 드래그
  • 왼손 본을 Shield Mount Parent에 드래그
중요

마운트 포인트 하위에 Sword_1, Sword_2, ..., Sword_6, shield_1, ..., shield_6 이름의 자식 오브젝트가 있어야 합니다. 없다면 각 Sword/Shield 3D 프리팹(Assets/Prefabs/Items/Equips/Sword/)을 해당 본의 자식으로 드래그해서 배치하고, 이름을 ItemData.asset의 equipObjectName과 동일하게 맞춰야 합니다. 배치 후 모두 SetActive(false)로 꺼두세요.

5-3. UI 씬 설정

Canvas 하위에 인벤토리 패널을 구성합니다.

  • Canvas 하위 → 새 Panel 생성 → 이름: InventoryPanel
  • Panel 하위 → Grid Layout Group 붙인 SlotContainer 오브젝트 생성
  • InventorySlotUi 프리팹 제작: Button + Image(아이콘) + TextMeshProUGUI(이름) 구성. 장착 하이라이트용 테두리 Image 하나 추가 (기본 비활성화)
  • 씬의 InventoryUi 컴포넌트(또는 Player GameObject)에 Panel Root, Slot Container, Slot Prefab 연결
  • 초기 상태에서 InventoryPanel을 비활성화

5-4. 상자 드랍 테이블 설정

씬에 배치된 각 Chest 프리팹을 선택합니다.

  • ChestInteractable 컴포넌트 → Drop Table Size 설정 (예: 2)
  • 각 Entry에 Sword2~6.asset 또는 Shield2~6.asset 드래그, Drop Chance 설정 (0.3~0.6 권장)
  • Player Inventory 필드는 비워둬도 됩니다 (Awake에서 자동 탐색)

5-5. 기본 장착 아이템 처리 (선택)

게임 시작 시 Sword1을 자동 장착하려면 플레이어 오브젝트에 아래 스크립트를 추가하거나, EquipmentManager.Awake() 안에 직접 처리합니다.

// EquipmentManager.Awake() 하단에 추가
// 기본 장착 아이템을 Resources 또는 인스펙터 참조로 지정
[Header("Default Equipment")]
[SerializeField] private ItemData _defaultSword;
[SerializeField] private ItemData _defaultShield;

private void Start()
{
    if (_defaultSword != null)  EquipItem(_defaultSword);
    if (_defaultShield != null) EquipItem(_defaultShield);
}
