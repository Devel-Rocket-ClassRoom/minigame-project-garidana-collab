플레이어 공격 강화 구조 구현 기록

Project Origin 2주차 핵심 루프인 퀘스트 진행 -> 레벨 업 -> 공격 강화 -> 다음 구역 진입을 위해, 플레이어 기본 공격이 레벨 구간에 따라 데미지, 판정 범위, 이펙트 색상을 바꾸도록 구조를 분리했습니다.

주요 파일PlayerAttackUpgrade.cs, PlayerAttack.cs, PlayerStats.cs
목표레벨 10 단위 기본 공격 강화와 VFX 연결 기반 마련

구현 요약

PlayerStats

레벨, 경험치, 기본 공격력 같은 플레이어 기본 수치를 관리합니다.

LevelChanged 이벤트를 추가해 레벨업 시 현재 레벨을 외부 컴포넌트에 알려줍니다.

PlayerAttackUpgrade

공격 강화 단계 전용 관리자입니다.

현재 레벨에 맞는 AttackStageData를 선택하고, 데미지 배율, 공격 범위, 이펙트 색상을 제공합니다.

PlayerAttack

공격 입력, 3단 콤보, 실제 타격 판정을 담당합니다.

공격 순간에 PlayerAttackUpgrade의 현재 강화 데이터를 읽어 데미지와 범위를 적용합니다.

실행 흐름

1. 레벨 상승 PlayerStats.LevelUp()에서 레벨과 기본 공격력이 증가합니다.
2. 이벤트 발행 LevelChanged 이벤트가 현재 레벨을 전달합니다.
3. 단계 갱신 PlayerAttackUpgrade.RefreshStage()가 레벨에 맞는 강화 단계를 고릅니다.
4. 공격 입력 PlayerAttack이 콤보 단계와 강화 데이터를 함께 사용합니다.
5. 결과 적용 데미지, 판정 범위, 파티클 색상이 현재 강화 단계에 맞게 적용됩니다.

강화 단계별 변화

Enhanced 3 Lv.30 x1.75 1.60 1.20 Gold 오크 구역 이상에서 강해진 기본 공격을 시각적으로 보여줍니다.

위 수치는 Assets/Scripts/Player/PlayerAttackUpgrade.cs의 기본 AttackStageData[] 값입니다. Unity 인스펙터에서 플레이어 오브젝트의 PlayerAttackUpgrade 컴포넌트를 통해 조정할 수 있습니다.

코드 변경 상세

1. PlayerStats는 레벨 변경만 알림

PlayerStats는 공격 강화 수치나 VFX를 직접 알지 않습니다. 대신 레벨업이 발생하면 LevelChanged 이벤트로 현재 레벨만 전달합니다. 

public event Action<int> LevelChanged;

public void LevelUp()
{
    _level++;
    IncreaseAttackPower(1f);
    LevelChanged?.Invoke(_level);
}

2. PlayerAttackUpgrade가 현재 강화 단계를 선택

레벨이 오르거나 컴포넌트가 켜질 때 RefreshStage()가 실행됩니다. 현재 레벨보다 해금 레벨이 낮거나 같은 단계 중 가장 마지막 단계를 현재 단계로 사용합니다. 

public void RefreshStage(int level)
{
    AttackStageData selectedStage = null;

    for (int i = 0; i < _attackStages.Length; i++)
    {
        AttackStageData stage = _attackStages[i];
        if (stage != null && level >= stage.UnlockLevel)
        {
            selectedStage = stage;
        }
    }

    _currentStage = selectedStage;
}

3. PlayerAttack이 실제 공격에 단계 데이터를 적용

기존 3단 콤보 구조는 유지했습니다. 데미지 계산식에 현재 강화 단계 배율을 추가했고, 공격 판정의 거리와 반경도 현재 단계 데이터에서 읽도록 변경했습니다. 

damage = AttackPower * ComboMultiplier * StageDamageMultiplier

hitCenter = playerPosition + forward * CurrentHitRange
hitRadius = CurrentHitRadius

현재 구조가 좋은 이유

  • PlayerStats가 공격 범위, 파티클, 콤보 같은 전투 세부 구현을 몰라도 됩니다.
  • PlayerAttack은 공격 판정과 데미지 적용에 집중할 수 있습니다.
  • PlayerAttackUpgrade만 보면 레벨별 강화 수치와 이펙트 정책을 한눈에 확인할 수 있습니다.
  • 퀘스트 보상으로 레벨이 한 번에 여러 개 오르거나 세이브로드로 특정 레벨에서 시작해도 현재 레벨 기준으로 단계가 다시 계산됩니다.
  • 나중에 스탯창 UI에서 현재 공격 강화 단계 이름을 읽어 표시하기 쉽습니다.

Unity 인스펙터 연결 방법

  1. 플레이어 오브젝트에 PlayerAttack, PlayerStats, PlayerAttackUpgrade가 붙어 있는지 확인합니다.
  2. PlayerAttackUpgrade의 Attack Stages 배열에서 단계별 수치를 조정합니다.
  3. 무기 또는 손 위치에 빈 오브젝트를 만들고, 해당 Transform을 Effect Spawn Point에 연결합니다.
  4. 각 단계의 Attack Effect Prefab에 사용할 파티클 프리팹을 연결합니다.
  5. 플레이 모드에서 레벨을 10, 20, 30, 40 이상으로 올린 뒤 공격 로그와 실제 판정 범위를 확인합니다.
이펙트 동작 방식

현재 구현은 공격 시작 시 현재 단계의 AttackEffectPrefab을 생성하고, ParticleSystem.main.startColor를 현재 단계 색상으로 바꾼 뒤 재생합니다. 프리팹을 아직 연결하지 않았다면 데미지와 범위만 적용되고 이펙트는 재생되지 않습니다.

남은 작업

  • 플레이어 프리팹 또는 씬 오브젝트에 PlayerAttackUpgrade 컴포넌트가 실제로 붙었는지 Unity 에디터에서 확인해야 합니다.
  • 각 단계별 파티클 프리팹을 연결해야 시각 효과가 보입니다.
  • 스탯창 UI에 현재 공격 강화 단계 이름과 수치를 표시하는 작업이 남아 있습니다.
  • 세이브로드 구현 시에는 레벨을 복원한 뒤 RefreshStage(Level)가 호출되는 흐름을 보장해야 합니다.
  • 실제 플레이 밸런스에 따라 데미지 배율과 공격 범위는 조정이 필요할 수 있습니다.
주의

현재 공격 강화 단계는 레벨 기준으로 자동 계산됩니다. 따라서 별도의 공격 강화 단계 값을 세이브 데이터에 저장하기보다는 레벨을 저장하고 로드 후 단계가 다시 계산되도록 만드는 편이 더 안전합니다.
